### Gemini Chatbot (complemento Typebot)

**Decisão estratégica:** Usar Gemini (LLM humanizado) vs Typebot (robótico)

**Implementado:**
- Migration 038: rename `typebot_status` → `bot_status`, novo `bot_type` (gemini|typebot|inactive)
- Tabela `ai_chatbot_configs`: system_prompt, model, temperature, end_keyword
- Hook `useAiChatbot`: fetchConfig, saveConfig, updateConversationBotContext
- Settings UI: AiChatbotSettings (prompt customizável, model selector, temperature slider)
- Edge Function `process-ai-messages`:
  - Busca config + histórico (bot_context JSONB)
  - Chama Gemini com sistema prompt + conversa
  - Detecta end_keyword → desativa bot
  - POST para Meta API (send_message)
  - Persiste histórico (últimas 20 msgs)
- Webhook integration: fire-and-forget async call
- End keyword handling: encerramento + transfer para atendente

**Fluxo completo:**
1. Cliente envia msg WhatsApp
2. Webhook → Edge Function (async)
3. Gemini processa com contexto
4. Resposta via Meta API para cliente
5. Histórico persistido em bot_context

## Decisões técnicas

- **Gemini vs Typebot:** LLM conversacional mais humanizado
- **Fire-and-forget webhooks:** nunca bloqueia 200 OK para Meta
- **bot_context JSONB:** histórico persistido, últimas 20 msgs para não encher tabela
- **pgcron vs N8N:** Supabase built-in, simpler + integrado
- **RSS + Google News:** scraper básico, expansível para APIs
- **Auto-create ideia:** workflow automático ao aceitar oportunidade
