### Decisões técnicas

- **Routing central**: Um número WhatsApp Meta API serve N clientes via phone mapping
- **Webhook flow**: Extract `from` → lookup `whatsapp_phone_mappings` → resolve `account_id` → pass to processMessage
- **Persistência**: `whatsapp_client_id` + `whatsapp_from_number` salvos em conversations
- **Hermes orchestrates**: Mantém mapping table atualizado (novo cliente → nova row)
- **Skill reutilizável**: Padrão pronto pra replicar em outros apps

### Problemas resolvidos

- Escalabilidade WhatsApp: Estava N integrações API (uma por cliente) → agora 1 número central
- Multi-tenancy routing: Phone lookup com RLS garante isolamento de dados
