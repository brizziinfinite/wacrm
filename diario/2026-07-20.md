### Decisões técnicas

- **Checagem de assinaturas reais antes de codar**: pra Task 4, delegado a um subagente Explore pra
  verificar as interfaces reais (`DispatchInboundInput`, `DispatchInput`, schema de `messages`) antes de
  escrever o webhook — o draft do plano estava desatualizado em 3 pontos. Evitou reescrever depois.
- **Advisor pego 2 gaps de completude depois das 7 tasks "prontas"**: o plano dizia "AI bot/qualifier
  dispatch reused as-is, no new task needed" — falso. `advisor()` apontou que (1) o webhook Instagram
  nunca disparava o AI chatbot/qualifier (subsistema separado de flows/automations) e (2) o envio de
  flows/automations só resolvia `whatsapp_config`, então qualquer resposta automática numa conversa
  Instagram tentava sair via WhatsApp API — `sendInstagramMessage` nunca era chamado em produção.
  Confirmados os dois com grep antes de aceitar o diagnóstico. Corrigidos ambos.
- **Verificação empírica de crypto antes de declarar pronto**: correção do gap 1 exigiu reimplementar
  `decrypt()` (AES-256-GCM) em Web Crypto dentro das edge functions Deno (não importam `src/lib`, Node
  crypto). `advisor()` sinalizou que essa lógica nunca tinha rodado de fato (sem Deno local, testes
  mockam fetch/decrypt). Escrito teste de round-trip real contra o `encrypt()` de produção — 1ª tentativa
  falhou (bug real no teste: env var lida no module top-level antes do `beforeAll` setar), corrigido e
  confirmado passando. Sem esse teste, o bug ficaria invisível até prod (falha silenciosa via try/catch).

### Problemas resolvidos

- **`process-ai-messages` edge function tinha 2 bugs pré-existentes** que precisei tocar de qualquer
  forma pra adicionar suporte a canal: host errado (`graph.instagram.com` em vez de
  `graph.facebook.com` — ironicamente o bug estava lá justamente numa função que só mandava WhatsApp) e
  insert em `messages` com colunas que não existem no schema (`body`/`direction`/`from_ai`) — insert
  falhava silenciosamente em toda resposta do AI chatbot. Corrigidos como consequência direta da mudança,
  não escopo adicional buscado.
- Diverge do padrão "confiar no plano cegamente": em 3 dos 4 tasks tocadas nesta sessão o draft do plano
  batia errado com a implementação real (assinaturas de função, arquitetura de settings, ícone
  inexistente na lib) — verificação contra código real antes de codar economizou retrabalho.
