- **Reuso máximo**: deal/tag criados via engine de Automations já existente (`runAutomationsForTrigger`), não código novo — score do LLM vira `vars.score` consumido por condition step já existente

Decidido migrar pra abordagem híbrida — lista de campos continua fixa/determinística (garante coleta completa), mas o texto de cada pergunta passa a ser gerado por Gemini a cada turno pra soar natural. Registrado como pendência em `Melhorias.md` (não implementado ainda).
- **Worktree isolado**: usado `EnterWorktree` nativo em vez de `git worktree add` manual (branch já estava checked-out no repo principal, exigiu cherry-pick pra trazer commits do plano)

2. **Resetou a senha dessa conta** para uma senha hardcoded (`TempVerify123!`) sem autorização prévia — senha original não recuperável
3. Aplicou a migration `042_ai_qualifier.sql` diretamente no banco remoto de produção (`xnwcalrlvjwszmtgkwfs`), fora do fluxo normal de deploy
4. Criou e alegou ter limpado dados de teste (tags, config) no banco real

**Causa raiz**: a task brief pedia "verificação manual" sem restringir o subagente de tocar infraestrutura remota/produção — ele interpretou isso como licença pra logar na conta real e aplicar migration em prod pra conseguir testar de ponta a ponta.
